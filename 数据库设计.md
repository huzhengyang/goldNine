## 一、集合列表（NoSQL）

### 1. users（用户集合）
```javascript
{
  _id: string,              // 用户ID
  uid: string,             // CloudBase 用户ID
  nickname: string,         // 昵称
  avatar: string,           // 头像URL
  phone: string,           // 手机号
  email?: string,          // 邮箱（可选）
  gender: string,          // 性别：男/女/保密
  age: number,            // 年龄
  city: string,           // 所在城市
  bio?: string,           // 个人简介
  level: string,          // 球技等级：青铜/白银/黄金/铂金/钻石/王者
  points: number,         // 积分
  createdAt: Date,        // 创建时间
  updatedAt: Date,        // 更新时间
}
```

### 2. news（新闻集合）
```javascript
{
  _id: string,              // 新闻ID
  title: string,            // 标题
  summary: string,          // AI摘要
  content: string,          // 正文内容（富文本HTML）
  coverImage: string,       // 封面图URL
  category: string,         // 分类：热门/赛事/技巧/视频
  source: string,          // 来源
  originalUrl: string,      // 原始URL
  keywords: string[],       // 关键词

  isPinned: boolean,       // 是否置顶
  status: string,         // 状态：草稿/已发布/已下架
  createdAt: Date,        // 创建时间
  updatedAt: Date,        // 更新时间
}
```

### 3. ai_analysis（AI分析记录集合）
```javascript
{
  _id: string,              // 分析ID
  userId: string,          // 用户ID
  videoUrl: string,         // 视频URL
  frames: string[],        // 关键帧URL数组

    grip: number,          // 握杆评分
    stance: number,        // 站位评分
    power: number,         // 发力评分

```

## 三、安全规则

### 基本原则
1. **读权限**：登录用户可读取所有公开数据
2. **写权限**：用户只能修改自己的数据
3. **管理权限**：管理员可管理所有数据
4. **商家权限**：商家可管理自己的球房、赛事、教练信息

### 示例规则（users 集合）
```javascript
{
  read: "auth.uid != null",  // 登录用户可读取
  write: "auth.uid != null && doc.uid == auth.uid",  // 只能修改自己的数据
}
```

### 示例规则（news 集合）
```javascript
{
  read: true,  // 所有人可读取
  write: "auth.uid != null",  // 登录用户可写（管理员需在后端验证）
}
```
