# GitHub 自动注册功能

## 功能说明

在 Kiro 账户管理页面，每个账户都有一个"注册 GitHub"按钮，点击后会创建注册任务，本地服务自动处理。

## 使用步骤

### 1. 启动本地服务

在本地电脑上运行：

```bash
cd goldnine/scripts
npm install
npx playwright install chromium
npm start
```

服务启动后会显示：

```
========================================
GitHub 自动注册服务（轮询版）
========================================
环境: goldnine-hk-8g3g5ijhec9df56e
轮询间隔: 5秒
========================================

服务已启动，等待注册任务...
```

### 2. 创建注册任务

在云端页面（手机或任何设备）：
1. 访问：https://goldnine-hk-8g3g5ijhec9df56e-1255523606.tcloudbaseapp.com/kiro
2. 登录账户管理
3. 展开要注册的账户卡片
4. 点击"注册 GitHub"按钮

### 3. 自动处理

本地服务会：
- 自动检测到新任务
- 打开浏览器注册 GitHub
- 从接码地址获取验证码
- 完成注册流程
- 自动更新状态

### 4. 查看结果

云端页面会自动轮询状态，注册成功后：
- 账户状态变为"已注册"
- 显示 GitHub 用户名

## 工作原理

```
云端页面
  ↓ 创建任务
云数据库（任务队列）
  ↓ 轮询获取
本地服务
  ↓ 打开浏览器
GitHub 注册页面
  ↓ 获取验证码
接码服务
  ↓ 更新结果
云数据库
  ↓ 轮询状态
云端页面显示结果
```

## 优势

- ✅ 无需暴露本地端口
- ✅ 支持 CORS 跨域
- ✅ 可以在手机上操作
- ✅ 任务持久化，可重试
- ✅ 实时状态更新

## 注意事项

- 本地服务需要保持运行
- 一个任务只处理一次
- 注册成功后自动标记
- 如遇人机验证需手动完成

## 常见问题

### Q: 点击按钮后没反应？

等待 3-5 秒，会弹出任务创建成功的提示。

### Q: 本地服务没检测到任务？

检查：
1. 服务是否正常运行
2. 控制台是否显示轮询日志
3. 网络是否正常

### Q: 注册失败？

可能原因：
1. 用户名已被占用
2. 接码地址无法获取验证码
3. GitHub 人机验证
4. 网络问题

解决：
- 修改邮箱前缀生成新的用户名
- 检查接码地址是否正确
- 手动完成人机验证
